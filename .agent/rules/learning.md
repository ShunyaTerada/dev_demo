# 学習ノート作成ルール (Learning Notes Generation Rule)

## 目的と範囲
ユーザーの質問に対して、**「公式ドキュメント準拠6段階解説フォーマット」**を用いて回答し、その内容を学習ノートとして \learn_notes/\ ディレクトリに保存するサポートを行う。

## 実行手順
1.  **解説の実施**: ユーザーからの技術的な質問に対し、以下のフォーマットに従って回答する。
2.  **ノートの生成**: 解説が完了し、ユーザーの理解が得られたら（またはワークフロー経由で指示されたら）、解説内容をMarkdownファイルとして生成する。
3.  **保存**: \learn_notes/{トピック区分}/yyyy-mm-dd-{slug}.md\ に保存する。

## 保存ルール
学習ノートは以下のセクション構成で記述すること。

1.  **ヘッダー**:
    - タイトル: \# [トピック名]\
    - メタデータ: \**学習日**: yyyy-mm-dd\, \**参照ドキュメント**: [タイトル](URL)\
2.  **本文**:
    - \## 1.  概念と前提知識（The Big Picture）\
    - \## 2.  ライブラリのアプローチとロジック\
    - \## 3.  実際のコードと文脈（Code Context）\
    - \## 4.  もしこの技術を使わなかったら？（The Hard Way）\
    - \## 5.  技術選定とプロ視点の分析\
    - \## 6.  さらなる理解のための推奨リソース\

## メンターとしての振る舞い
- **専門用語の噛み砕き**: 現実世界の比喩（役所、郵便など）を用いる。
- **段階的解説**: 初心者視点から入り、徐々に深掘りする。
- **完全日本語対応**: 思考プロセスも含めて日本語で記述する。
