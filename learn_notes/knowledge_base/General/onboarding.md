# Onboarding（オンボーディング）

**学習日**: 2025-11-24
**参照ドキュメント**: [Storybook Tutorials - Intro to Storybook](https://storybook.js.org/tutorials/intro-to-storybook/react/ja/get-started/)

## 1. 🧠 概念と前提知識（The Big Picture）
- **定義と分類**: 「オンボーディング」は元々人事用語で、新入社員を組織に定着させるための「受け入れ研修」や「導入支援」を指します。ソフトウェアの文脈では、**「初めてそのツールを使うユーザーが、迷わずに使い方を理解し、スムーズに利用開始できるようにするためのガイド機能」**を指します。
- **現実世界の比喩**:
    - **飛行機や船**: 乗客が搭乗すること（On-board）が語源です。
    - **ゲーム**: 最初に操作方法を教えてくれる「チュートリアルステージ」のことです。いきなりラスボス戦に放り出されるのではなく、スライムで練習させてくれる親切設計です。
- **実用的な文脈**: CLIツールやSaaS製品では、初期設定ウィザードや、画面上のツアー（「ここをクリックするとこれができます」という吹き出し）として実装されます。

## 2. 🛠 ライブラリのアプローチとロジック
StorybookのCLI（コマンドラインツール）におけるオンボーディングは、以下のロジックで動いています。
1.  **検出**: プロジェクトに初めてStorybookがインストールされたことを検知します。
2.  **提案**: 「使い方がわからないでしょうから、サンプルを用意しましょうか？」と聞いています（これが画面の `Yes: Help me with onboarding` です）。
3.  **生成**: `Yes` を選ぶと、空っぽの設定ファイルだけでなく、**実際に動くサンプルのコンポーネント（Button, Header, Pageなど）と、それに対応するストーリーファイル**を自動生成してくれます。

## 3. 📝 実際のコードと文脈（Code Context）
画面に出ている選択肢の文脈を分解します。

```bash
? New to Storybook? 
# Storybookは初めてですか？

> Yes: Help me with onboarding
# はい: オンボーディング（導入支援）をお願いします
# → 結果: サンプルファイル（stories/Button.stories.tsなど）が作成され、使い方の解説が表示されます。

  No: Skip onboarding & don't ask again
# いいえ: オンボーディングをスキップして、二度と聞かないでください
# → 結果: 最小限の設定ファイルのみが作成されます。玄人向けです。
```

## 4. 🚫 もしこの技術を使わなかったら？（The Hard Way）
もしオンボーディング（`No`）を選択した場合：
- **真っ白な状態**: Storybookは起動しますが、表示するコンポーネントが一つもありません。
- **手探り**: 自分でドキュメントを読み、ゼロから `MyComponent.stories.ts` というファイルを作成し、正しい構文で記述する必要があります。これは初心者にはハードルが高い「死の谷」になりがちです。

## 5. ⚖️ 技術選定とプロ視点の分析
- **推奨**: 初めてStorybookを使う場合は、**迷わず `Yes` を選んでください**。
- **メリット**: 生成されたサンプルコードを見ることで、「あ、こういう風に書けばいいんだ」という**正解の型（ベストプラクティス）**をコードベース内で直接確認できます。
- **その後**: 慣れてきたら、生成されたサンプルファイル（`src/stories` フォルダなど）は削除して構いません。
